Get Started

To be ble to create applications using Wikidata, we connected Wikidata with Mavo, an HTML templating langauge that allows users to author applications using HTML only, without writing any JavaScript code. In addition, we connected to Shapir, a library that allows users to build Mavo applications that connects Wikidata with other websites (e.g. YouTube, NY Times, etc). In order to use Mavo, you first need to download or link to its JS and CSS files and include them in your page. If you have never done this before, you could put both in the <head> section. In addition, in the <body> you will need to define the Mavo application, like this: 

<head>
	<script rel="stylesheet" src="https://get.mavo.io/mavo.css"></script>
	<script src="https://get.mavo.io/mavo.j"></script>
	<script type="module" src="https://shapir.org/mavo-shapir.js"></script>
</head>
<body>
	<div mv-app mv-source="shapir" mv-source-service="wikidata" mv-source-language="en">
		<!-- add your HTML code here -->
	</div>
</body>

Read a Wikidata item

If you are intersted in getting an information about a specific Wikidata item (e.g. United States of America (Q30)), then you can specify that using mv-source-id= "Q30" 

<div mv-app mv-source="shapir" mv-source-service="wikidata" mv-source-language="en" mv-source-id="Q30">
    <!-- add your HTML code here -->
</div>

Read a list of Wikidata items

If you are intersted in reading information about a list of Wikidata items (e.g. List of national memorials of the United States), then you can specify that using mv-source-instance-of="National Memorial of the United States".

<div mv-app mv-source="shapir" mv-source-service="wikidata" mv-source-language="en"
    mv-source-instance-of="National Memorial of the United States">
    <!-- add your HTML code here -->
</div>

  • You can actually use any Wikidata properties to query Wikidata. In the above example we queried Wikidata using the instance of property. You can use any Wikidata property using the mv-source-* attribute. For example, if you want to get a list of writers who were born in New York City, you can use occupation and place of birth in the following way: mv-source-occupation= "writer" and mv-source-place-of-birth="New York City".
  • You should also add property="items" mv-multiple>

Spaces in properties need to be replaced with a hyphen - For example, instance of will be instance-of 

<div mv-app mv-source="shapir" mv-source-service="wikidata" mv-source-language="en"
    mv-source-occupation="writer" mv-source-place-of-birth="New York City">
    <!-- add your HTML code here -->
</div>

Specify Number of Results

To specify the number of results returned by a search, you add mv-source-numberOfItems to the mv-app div. For example, if you want to show only 10 writers from New York City, then you can do it this way.


Example

<div mv-app mv-source="shapir" mv-source-service="wikidata" mv-source-language="en"
    mv-source-occupation="writer" mv-source-place-of-birth="New York City" mv-source-numberOfItems="10">
    <!-- add your HTML code here -->
</div>

Show Wikidata simple properties

You can display and show Wikidata properties in your application by using the property="" attribute from Mavo.


For every item, we offer the follwing common properties:

  • label: the label of the item
  • description: the description of the item
  • aliases: aliseas of the item
  • itemID: the ID of the item (e.g. Q30)
  • itemURL: the URL of the item page on Wikidata (e.g. https://www.wikidata.org/wiki/Q30)
  • Other properties: all other properties provided by the item, you can using it by remove the space between words and capitlize the words after the space. For example instace of will become property="instanceOf"

The follwing is a simple example that displays information about Herman Melville.

HTML

<div mv-app mv-source="shapir" mv-source-service="wikidata" mv-source-language="en" mv-source-id="Q4985">

    <img property="image" />
    <h1 property="label"><h1>
    <p property="description"><p>
    <time property="dateOfBirth"><time>
    ....
</div>

You can also use the same code above to show a list of writers from New York City. The only thing that you will need to add is property="items" mv-multiple, like the below example. Feel free to check the Mavo documnetation to know more about mv-multiple

HTML

<div mv-app="main" mv-source="shapir" mv-source-service="wikidata" mv-source-langugae="en"
  mv-source-occupation="writer" mv-source-place-of-birth="New York City" mv-source-numberOfItems="10">

    <div property="items" mv-multiple>
        <img property="image" />
        <h1 property="label"><h1>
        <p property="description"><p>
        <time property="dateOfBirth"><time>
    </div>

</div>

Show properties with values as items

To show a property with an item as its value, for example, if we want to also display the country of citizenship for Herman Melville in the application above, property="countryOfCitizenship" and then use property="label" to read the label of the property's value, like this: 


<div mv-app mv-source="shapir" mv-source-service="wikidata" mv-source-language="en" mv-source-id="Q4985">
    <img property="image" />
    <h1 property="label"><h1>
    <p property="description"><p>
    <time property="dateOfBirth"><time>
    <div property="countryOfCitizenship">
        <!-- the value of this is United States of America (Q30), you can use any of its properties-->
        <p property="label"><p>
        <p property="description"><p>
        <div property="capital">
            <p property="label"><p>
        </div>
    <div>
    ....
</div>

Propertie created to hold other website's data

Using Wikxhibit you display information or data about Wikidata items from other websites. For example, if you want to build a web page about Lady Gaga that shows information about her from Wikidata but also display and play her tracks from Spotify, then you can use the Spotify artist ID from Wikidata. The way you use it in Wikxhibit is by removing the word ID, capetilize the first letter of the word Artist and remoe the space between them SpotifyArtist and use it as a property property="SpotifyArtist". This property will contains an object of different information about Lady Gaga from Spotify (e.g. album and track).


HTML

<div mv-app mv-source="shapir" mv-source-service="wikidata" mv-source-id="Q19848">

    <img property="image" />
    <h1 property="label"></h1>
    <p property="description"></p>
    ...
    <div property="SpotifyArtist">
        <div property="albums" mv-multiple>
            <img property="image" />
            <h5 property="name"></h5>
            <p property="numTracks"></p>
            <p property="datePublished"></p>
        </div>

        <div property="tracks" mv-multiple>
            <iframe property="identifier"
                src="https://open.spotify.com/embed/track/[identifier]"></iframe>
        </div>
    </div>

</div>
